# 文档格式规范

**所属合集**: IPD工具包改造计划

## 标题层级

- `#` — 文档标题（每篇仅一个）
- `##` — 主要章节
- `###` — 子章节
- `####` — 三级子章节（谨慎使用）

## 交叉引用语法

在同一合集中引用其他文档时，使用相对Markdown链接：

```markdown
详见[转化路线图](01-转化路线图.md)中的阶段规划。
```

引用IPD融合指南时，使用docs根目录的相对路径：

```markdown
参见IPD融合指南[第三章](../../IPD与敏捷开发深度融合的软件产品开发流程与研发管理平台搭建指南.md)。
```

引用宪法原则时，使用编号：

```markdown
本指南与**原则III（敏捷-门径治理）**对齐。
```

## 术语规范

- 每篇文档中，专业术语首次出现时使用"中文名（English Term）"格式，例如：
  "技术评审（Technology Review, TR）"
- 后续使用缩写即可，例如"TR1门径"
- 所有4篇文档中保持缩写一致
- 完整术语表参见[术语表](glossary.md)

## 元数据前缀

每篇文档开头应包含简要元数据块：

```markdown
**文档**: IPD转化路线图
**所属合集**: IPD工具包改造计划
**状态**: 草稿
**日期**: 2026-06-06
```

## 表格

- 使用GFM表格语法，带对齐标记
- 保持表格可读性：不超过6–8列
- 空单元格使用`—`而非留白

## 代码块

代码块（用```围起）仅用于：
- 结构性示例（如目录树、JSON结构）
- 说明性配置模式（描述性，非可执行）

根据"纯描述性"范围约束，不应出现可执行代码、脚本或命令。

## 英文原文链接

每篇中文文档末尾应添加"英文原文"链接，指向对应的英文文档：

```markdown
---
[英文原文 / English Original](../01-transformation-roadmap.md)
```

## 文件命名

- 所有文件使用小写加连字符格式：`01-转化路线图.md`
- 数字前缀表示阅读顺序（01→04）
- 术语表和贡献规范无数字前缀
- 中文文档与英文文档使用相同的数字前缀以便对照
