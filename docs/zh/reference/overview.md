# CLI 参考

Specify CLI（`specify`）管理规范驱动开发的完整生命周期 — 从项目初始化到工作流自动化。

## 核心命令

用于创建和管理 vibe-ipd 项目的基础命令。初始化新项目，包含必要的目录结构、模板和脚本。验证系统是否已安装所需工具。查看版本和系统信息。

[核心命令参考 →](core.md)

## 集成

集成功能将 vibe-ipd 连接到你的 AI 编码智能体。每个集成会为特定智能体设置相应的命令文件、上下文规则和目录结构。每个项目一次只能激活一个集成，你可以随时切换。

[集成参考 →](integrations.md)

## 扩展

扩展为 vibe-ipd 增加新能力 — 领域特定命令、外部工具集成、质量门禁等。它们通过目录被发现，可以独立安装、更新、启用、禁用或移除。多个扩展可以在同一项目中共存。

[扩展参考 →](extensions.md)

## 预设

预设自定义 vibe-ipd 的工作方式 — 覆盖命令文件、模板文件和脚本文件，而无需更改工具本身。它们让你可以强制执行组织标准、使工作流适配你的方法论，或本地化整个体验。多个预设可以按优先级顺序叠加以分层定制。

[预设参考 →](presets.md)

## 工作流

工作流将多步骤的规范驱动开发过程自动化为可重复的序列。它们将命令、提示、Shell 步骤和人工检查点链接在一起，支持条件逻辑、循环、分发/汇聚，以及从中断点精确暂停和恢复的能力。

[工作流参考 →](workflows.md)
