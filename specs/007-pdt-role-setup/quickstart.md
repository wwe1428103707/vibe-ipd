# Quickstart: Using IPD Role Agents

## Setup

Role agent skills are installed as `.claude/skills/vipd-agent-assign-*/SKILL.md` files.
After adding new skill files, run `/reload-skills` in Claude Code to register them.

## Basic Usage

### Single Role Invocation

```
/vipd-agent-assign-lpdt Review TR1 gate evidence for feature 007
```

The agent will respond as LPDT/RTE, assessing gate readiness.

```
/vipd-agent-assign-po Assess feature validity for proposed user auth feature
```

The agent will respond as PO, evaluating business value and backlog priority.

### Product Trio Discovery Workflow

Run a structured feature validation with all three discovery-track roles:

1. **PO assessment**: `/vipd-agent-assign-po` — evaluate market need and success criteria
2. **Architect assessment**: `/vipd-agent-assign-architect` — evaluate technical feasibility
3. **Synthesize**: /vipd-clarify or manual review of combined findings

### Gate Review

```
/vipd-agent-assign-lpdt Conduct TR2/TR3 gate review for feature 007
/vipd-agent-assign-qa-lead Assess TR4 quality evidence
```

### Combined Role (Small Teams)

For teams of 3-4 where one person fills multiple roles:

```
/vipd-agent-assign-lpdt-po Review gate evidence and backlog priority
```

The agent will respond with both LPDT and PO perspectives, using conflict resolution rules.

## RACI-Annotated Tasks

When tasks are generated by `/vipd-tasks`, look for `RACI:` annotations:

```text
- [ ] T001 Create user model [P1] [Story1] src/models/user.py RACI: R=architect, A=architect
- [ ] T002 Write API tests [P2] [Story2] tests/test_api.py RACI: R=qa_lead, A=qa_lead
```

## Role Responsibilities Quick Reference

| Command | Role | Best For |
|---------|------|----------|
| `/vipd-agent-assign-lpdt` | LPDT/RTE | Gate reviews, dependency management, escalation |
| `/vipd-agent-assign-po` | PO | Feature validation, backlog priority, acceptance criteria |
| `/vipd-agent-assign-architect` | Architect | Technical feasibility, architecture compliance |
| `/vipd-agent-assign-qa-lead` | QA Lead | Quality metrics, test strategy, DoD |
| `/vipd-agent-assign-devops-lead` | DevOps Lead | CI/CD review, deployment readiness |
| `/vipd-agent-assign-lpdt-po` | LPDT+PO (combined) | Small-team single-person gate reviews |
| `/vipd-agent-assign-devops-qa` | DevOps+QA (combined) | Small-team quality+operations reviews |

## Example: Full Discovery Session

```
You: /vipd-specify Add user profile management
      → Spec created at specs/008-user-profile/spec.md

You: /vipd-agent-assign-po Review spec 008-user-profile
      → PO: Feature valid, P2 priority, recommend proceeding

You: /vipd-agent-assign-architect Review spec 008-user-profile for feasibility
      → Architect: Feasible with existing auth system, low complexity

You: /vipd-clarify 
      → Resolves ambiguity, updates spec

Result: Feature cleared for delivery backlog (TR1 passed)
```
