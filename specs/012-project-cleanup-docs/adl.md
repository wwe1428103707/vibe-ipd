# Architecture Decision Log: Project Cleanup & Documentation Rebrand

**Feature**: [spec.md](spec.md) | **Date**: 2026-06-07 | **Phase**: TR2/TR3 Plan & Design

## ADL-001: Full README Rewrite Over Incremental Patch

**Status**: ✅ Accepted

**Context**: The root README currently identifies primarily as "Spec Kit" with vibe-ipd as a secondary reference. Options were to incrementally patch references or do a full rewrite.

**Decision**: Full rewrite of root README.md with a new structure centered on vibe-ipd's identity.

**Rationale**:
- Incremental patching would leave structural gaps and read incoherently
- Full rewrite allows proper storytelling: what → why → how → differentiators
- README is a fresh-slate document; no backward-compatibility concerns

**Consequences**:
- README structure changes completely — link anchors from external references may break
- Must include acknowledgments section for speckit attribution

## ADL-002: Grep-Based Validation Over Manual Review

**Status**: ✅ Accepted

**Context**: Verifying that all stale branding references have been updated requires a repeatable, quantitative approach.

**Decision**: Use `grep -r` with specific patterns as the primary validation mechanism.

**Rationale**:
- Quantitatively verifiable per SC-002 and SC-003
- Can be automated in CI and repeated on demand
- Clear pass/fail criteria: zero hits for stale patterns (excluding attribution)

**Consequences**:
- Validation patterns must be documented and agreed upon before implementation
- Attribution exclusions must be clearly defined

## ADL-003: Preserve speckit Attribution in Dedicated Section

**Status**: ✅ Accepted

**Context**: The project is derived from speckit (Spec Kit). Over-rebranding could erase this history.

**Decision**: Include an "Acknowledgments" or "Origin" section in the README that explicitly credits speckit as the upstream project.

**Rationale**:
- Maintains project genealogy and legal/ethical attribution
- Allows clean rebranding everywhere else without erasing history
- Provides context for users migrating from speckit

**Consequences**:
- The attribution section becomes the single allowed location for speckit references
- Must clearly distinguish "upstream speckit" vs. "vibe-ipd" in all documentation
