# Feature Specification: Editorial Gate Report

**Feature Branch**: `020-editorial-report`

**创建日期**: 2026-06-09

**状态**: Draft

**用户描述**: 使用 TailwindCSS 重写门控历史报告页面为现代编辑风格报纸布局，包含 Hero story、多栏排版、引言摘录、工作流记录展示。

## 用户场景

### 场景 1 — 报纸风格门控报告 (P1)
读者打开 HTML 报告时看到一份排版严谨的"数字报纸"：顶部报头含期号/日期，头版故事展示最新的门控状态，多栏区域组织不同门控阶段的历史记录，引言卡片高亮关键数据。

### 场景 2 — 工作流记录展示 (P2)
报告中增加工作流指标区域：并行任务数、启动的 Agent 数、阶段完成率、执行时长等。

## 功能需求

- **FR-001**: 报告使用 TailwindCSS 样式（CDN 内联加载），纯静态自包含 HTML
- **FR-002**: 布局包含报头/导航区、Hero 头版故事、多栏内容区、引言摘录、CTA/订阅区、页脚
- **FR-003**: 颜色：黑白灰为主 + 单色点缀（深蓝/酒红/哑金），细线分隔，无阴影
- **FR-004**: 三档响应式：移动端单栏、平板双栏、桌面多栏（12列网格）
- **FR-005**: 展示工作流指标：并行 Agent 数、任务分布、阶段通过率、执行时长
- **FR-006**: 门控时间线以"头版文章"和"专栏文章"形式呈现，而非表格
- **FR-007**: 数据通过静态 JSON 注入，无需外部请求

## 成功标准

- **SC-001**: 报告在 Chrome/Firefox/Edge 中渲染一致
- **SC-002**: 从打开到内容可视 <1秒（纯静态 HTML）
- **SC-003**: 工作流指标区域至少包含 4 项指标

## 关键实体

- 门控记录 (GateRecord), 工作流统计 (WorkflowMetrics)
